Annotator Guide

Everything you need to know to evaluate STT data samples accurately and efficiently.

🚀 Getting Started

  1. 1 Log in with the credentials provided to you (e.g. Annotator1 / pass1).
  2. 2 You will see your assigned sample folders on the left panel. Double-click a folder to open it.
  3. 3 Inside a folder you will see audio files. Single-click any audio file to load it.
  4. 4 Listen to the audio, read the transcripts, fill in the audit form on the right, then click Save.
  5. 5 Repeat for every file. The progress bar at the top tracks how many you have completed.

🖥️ Interface Overview

The screen is split into two panels:

  • Left panel — File Browser: Browse folders and audio files. Files with a ✓ have already been annotated.
  • Right panel — Player & Audit Form: Audio player, transcript display, quality metrics, and the evaluation form.

Dual Transcript

Two transcripts are shown side by side:

  • Ground Truth — the reference transcript from the dataset.
  • ElevenLabs STT — the transcript generated by the ElevenLabs speech-to-text engine.

You can click on individual words in either transcript to mark them as incorrect. Clicked words turn red and are automatically added to the "Incorrect in …" fields below.

Transcript Correctness

For each transcript (Ground Truth and ElevenLabs STT), indicate whether it is correct or incorrect compared to what you hear in the audio.

  • Select Yes if the transcript accurately represents the spoken audio.
  • Select No if there are errors.
Important: If both transcripts are marked as incorrect, a text field will appear asking you to type the correct transcript. Please listen to the audio carefully and write what you hear as accurately as possible.

Issue Flags

Check any that apply:

  • Wrong Language — audio is in a different language than expected.
  • Missing Words — transcript omits words that are clearly spoken.
  • Garbled/Nonsense — transcript contains meaningless text.
  • Excessive Silence — long pauses or silence dominate the audio.
  • Proper Noun Errors — names of people, places, or brands are wrong.
  • Punctuation Issues — punctuation significantly alters meaning.

Incorrect Words

Either click words in the transcript to mark them, or type them manually into the two "Incorrect in …" fields. Separate multiple words with commas.

💾 Saving & Progress

  • Click Save after completing the audit for each file. The button briefly turns green to confirm.
  • A ✓ indicator appears next to saved files in the file list.
  • The progress bar in the header shows how many files you have annotated out of the total.
  • Copy TSV copies all the evaluation data for the current file as a tab-separated row, useful for pasting into spreadsheets.
Don't forget to save! Navigating to another file before saving will discard your unsaved work for the current file.

💡 Tips for Efficient Annotation

  • Use the arrow keys to quickly move through files without reaching for the mouse.
  • Listen to the audio at least once fully before filling in the form.
  • If a file is clearly unusable (wrong language, corrupt audio), mark it quickly and move on — don't spend extra time.
  • Use bookmarks (⭐ button) to save your place if you need to take a break.
  • The WER badge next to each file in the list gives you a quick preview — high WER files are more likely to need careful review.

❓ FAQ

What if I can't understand the audio at all?

Mark Audio Quality as Unusable and Overall Quality as Unusable. Add a note explaining why (e.g., "too much static", "speaker is inaudible").

What if the Ground Truth and ElevenLabs transcripts are different but both seem correct?

This can happen with minor variations (e.g., numbers as words vs. digits). Mark both as correct and note the difference in the Notes field.

Can I go back and edit a previously saved annotation?

Yes. Click on the file again — your saved annotation will be loaded back into the form. Make your changes and click Save again to overwrite.

What if I accidentally close the browser?

Your session is preserved. Log in again and you will return to where you left off. All previously saved annotations are safe on the server.
